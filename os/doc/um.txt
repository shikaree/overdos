÷verDOS - User's manual

Booting: load the ÷verDOS bootfile on a Algorithmics P4032 mips computer by 
typing, when connected to the mips, boot /.../bootfile, where ... is the path 
to the ÷verDOS bootfile. Then type "g".

Using ÷verDOS is centered around the command interpreter B‰rs, or rather the
extended version Visky. B‰rs and Visky are described in the ÷verDOS 
application documentation.

Focus switch: the process that is allowed to perform i/o is said be in focus. 
You select wich process that shall be in focus by pressing ` on your keyboard,
the button next to backslash. 

You can always start a new Visky (command interpreter) by pressing ', 
the button left of return. This can be very handy if you accidentially kill
your command interpreter.

It's strongly recommended that you run ÷verDOS in a terminal program that 
supports ansi colours, such as Eterm. Running ÷verDOS in a terminal that does
not support ansi colours, such as xterm, will work too but is not recommended.


Technical Specifications

Number of supported processes decided at compile time, theoretical maximum of 
2 billion processes (but is limitided by availible memory).

Theoretical maximum number of mailboxes 2 billion.

Supports full vt-100 emulation.

Partial Spim compatibility.
